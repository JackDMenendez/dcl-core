# v0.2.0 Release Notes

**Released:** 2026-06-07
**Prior version:** v0.1.0 (2026-05-22, doi:10.5281/zenodo.20350952)
**Tag:** v0.2.0
**Semver impact:** MINOR (pre-1.0).  The `core3d` two-frame naming
retrofit renames several public symbols, but every old name still
resolves through a backward-compatible deprecation shim, so existing
callers do not break.  Pre-1.0 semver still applies -- callers should
expect MINOR bumps to break them, and should migrate off the
deprecated aliases before the first MAJOR after 1.0.
**DOI:** [10.5281/zenodo.20586191](https://doi.org/10.5281/zenodo.20586191)

## Why v0.2.0

Two things make v0.2.0 a release rather than a patch:

1. **The cross-validation layer is complete.**  All four
   `tests/test_cross_validation.py` tests now pass (v0.1.0 shipped
   them as `@pytest.mark.skip`).  This was the framework's stated v1.0
   gap; closing it means the `core` <-> `core3d` relationship is now
   pinned by executable tests rather than prose.  We are cutting this
   as **v0.2.0, not v1.0.0** -- the public API is deliberately kept
   under the pre-1.0 "expect breakage" signal for one more cycle (GPU
   kernels and inter-session coupling are still ahead and may move the
   surface).

2. **`core3d` adopts the two-frame naming convention.**  Public
   symbols are renamed so the *name* says what the object IS in the
   lattice's own mathematics, with a `# physics:` comment carrying the
   existing-physics correspondence.  All renames ship
   backward-compatible shims (see *Deprecated* below).

The `dcl_core.core` submodule (the Paper~I port) is frozen and
untouched.

## Public-API changes (vs v0.1.0)

### Added

- `dcl_core.core3d.TokenResidual` -- canonical name for the
  fractional-bit-carry quantiser (was `BresenhamResidual`).
- `DiscreteCausalSession` state properties `N_RGB`, `N_CMY`,
  `phi_RGB`, `phi_CMY` -- canonical lattice-intrinsic names for the
  per-chirality token counts and phase fields.
- `DiscreteCausalSession.amplitude(component=...)` -- `component`
  takes `"RGB"` / `"CMY"` (the legacy `"R"` / `"L"` values and the
  `chirality=` keyword are still accepted).
- `TokenResidual` / `BipartiteLattice` gain `dp_min` (alias of
  `epsilon_P`) and `coordination` respectively.

### Changed

The `core3d` two-frame naming retrofit renamed the following public
symbols.  **Every old name still resolves** (see *Deprecated*); these
are listed here because the canonical name a new caller should use has
changed:

- `BresenhamResidual` -> `TokenResidual` (class).
- session state `N_R` / `N_L` -> `N_RGB` / `N_CMY`.
- session state `phi_R` / `phi_L` -> `phi_RGB` / `phi_CMY`.
- `DiscreteCausalSession.amplitude(chirality=...)` ->
  `amplitude(component=...)`.
- `BresenhamResidual.epsilon_P` -> gains `dp_min` as the canonical
  alias.

See `docs/design/03_naming_convention.md` for the full rule and
glossary.

### Deprecated

All of the following resolve today and emit a `DeprecationWarning`
where a keyword is involved; they will be removed no earlier than the
first MAJOR release after 1.0:

- `BresenhamResidual` (class alias of `TokenResidual`).
- `N_R` / `N_L` / `phi_R` / `phi_L` (property aliases).
- `amplitude(chirality=...)` and the `"R"` / `"L"` component values.
- The old keyword names on `from_arrays`, `amplitude`, and
  `TokenResidual.quantise`.

### Removed

(None -- every v0.1.0 public name still resolves.)

## Internal changes

- **Cross-validation tests landed (all 4 now pass).**  The suite is
  now `232 passed, 26 xfailed, 0 skipped` (v0.1.0: `228 passed, 4
  skipped, 26 xfailed`).  The 26 xfails remain pre-existing Paper~I
  port drift.  The four tests were respec'd to the regime where the
  two engines are actually comparable -- see CLAUDE.md's
  cross-validation triage for the engine-protocol-mismatch reasoning:
  - `test_conservation_invariants_agree` -- each engine conserves its
    own A=1 invariant.
  - `test_free_propagation_matches_in_large_N_limit` -- reframed as a
    `core3d` self-convergence test: integer-token density tracks the
    continuous amplitude it quantises with L2 error
    `O(sqrt(n_sites)/n_units)`.
  - `test_two_body_orbit_locks_in_both_cores` (`slow`) -- each engine
    settles to a stable interior radius in a fixed Coulomb well;
    pins the >0.2 fixed-lattice divergence as a tripwire.
  - `test_arnold_tongue_locations_agree` (`slow`) -- single-session
    orbital resonance; each engine shows a frequency-dependent
    lock-in, and the tightest-lock `omega` is required to differ.

## New design docs / notes

- **`docs/design/03_naming_convention.md`** -- the two-frame naming
  convention for `core3d` (the name says what the object IS; a
  `# physics:` comment names the existing-physics correspondence) plus
  the symbol glossary.  Enforced on diffs by the
  `.claude/agents/physics-naming-reviewer.md` agent.
- **`notes/color_structure.md`** (DRAFT) -- geometric RGB gives only
  Z_3; real SU(3) colour lives on a separate C^3 colour-memory factor
  (Paper~II result).  Implementing it in `core3d` is what would first
  couple `dcl_core` to Paper~II.
- **`docs/data_deposit_and_provenance.md`** -- per-experiment
  provenance manifests; one Zenodo `dataset` record per paper; heavy
  data deposited (not bundled into the software DOI), `isDerivedFrom`
  the released engine version.

## Reproducibility

```sh
git clone https://github.com/JackDMenendez/dcl-core.git
cd dcl-core
git checkout v0.2.0
python -m venv .venv
source .venv/bin/activate   # POSIX
# .venv\Scripts\activate.bat # Windows cmd
pip install -e ".[dev]"
pytest tests/ -v
```

The two `slow`-marked cross-validation tests
(`test_two_body_orbit_locks_in_both_cores`,
`test_arnold_tongue_locations_agree`) run by default; deselect with
`-m "not slow"` for a faster check.

## What is NOT in v0.2.0

Unchanged from v0.1.0's deferral list -- still ahead:

- **GPU backend kernels.**  CuPy allocators are wired; `RawKernel`
  bodies remain TODO strings.
- **Pairwise multi-session interactions** (Coulomb / gauge /
  emission coupling between sessions).  The richer two-session
  Arnold-tongue (frequency-ratio x coupling) needs this.
- **Complex `TokenResidual` carry.**  Real carry only; the
  complex-carry hypothesis is parked for Paper~III's min-Δp
  experiments (`notes/bresenham_residual_design.md`).
