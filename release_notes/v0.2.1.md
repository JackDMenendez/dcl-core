## Why v0.2.1

A single additive feature: the **`prob_floor`** parameter on
`dcl_core.core.CausalSession`.  It is the continuous engine's
`delta_p_min` knob, and `dcl-delta-p-min`'s 4-cell grid (the `core`
column of `exp_09_dp_min_floor`) is blocked without it.  This release
exists to give that downstream investigation a pinnable artifact
(`dcl_core @ git+...@v0.2.1`) so its Phase 2 can run.

This is **not** the v1.0.0 cut.  v1.0.0 (the formal API freeze) remains
gated on `dcl-delta-p-min` finishing and co-releases with Paper III;
v0.2.1 is an interim release toward it.

## Public-API changes (vs v0.2.0)

### Added

- **`dcl_core.core.CausalSession(..., *, prob_floor=None)`** -- a
  keyword-only minimum per-site probability.

  Per-site probability is `p(x) = |psi_R(x)|^2 + |psi_L(x)|^2`.  At
  end-of-tick, any node with `0 < p(x) < prob_floor` has both spinor
  components scaled by the real factor `sqrt(prob_floor / p(x))`, so
  `p(x) == prob_floor` exactly with phase (and the `psi_R/psi_L`
  ratio) preserved; the joint state is then renormalised so
  `sum(p) == 1` (A=1 preserved).  Nodes with `p(x) == 0` carry no
  phase and stay at zero -- the floor is the minimum *non-zero*
  probability quantum (the continuous-engine analogue of `core3d`'s
  `1/N = delta_p_min`).

  Validated to lie in the open interval `(0, 1)`; `None` (default)
  clamps nothing.

### Changed / Removed / Deprecated

(None.  `prob_floor=None` is bit-for-bit identical to v0.2.0.)

## Internal changes

- `CausalSession` is no longer a byte-for-byte Paper~I port: it carries
  this one sanctioned, backward-compatible addition.  The naming-
  convention exemption for `dcl_core.core` is unchanged.

## Tests

`pytest tests/` reports **252 passed, 26 xfailed, 0 skipped** (v0.2.0:
232 passed).  The +20 are the new `prob_floor` suite:

- `tests/test_prob_floor_a1_consistency.py` -- the spec-named contract
  test: validation, backward-compat (None == old, bitwise), per-site
  clamp to floor exactly, phase + R/L ratio preserved, zero-site stays
  zero, end-to-end floor engagement, and A=1 after every tick across
  `prob_floor` in {1e-10 ... 0.05} (incl. massless).

The 26 xfails remain pre-existing Paper~I port drift.

## New experiments

- **`exp_03_prob_floor_diag`** (ACTIVE) -- sweeps `prob_floor` across
  decades on one fixed wavepacket and records `max_unity_residual`
  (stays ~1e-16 -> floor never breaks A=1), whether the floor engaged,
  support size, and rms radius.  The `core` column of
  `dcl-delta-p-min`'s `exp_09` in miniature.  Writes
  `data/exp_03_prob_floor_diag.{npy,log}`.

## Docs

- `docs/reference/api.md` gains a `prob_floor` section (semantics,
  the `delta_p_min -> prob_floor` mapping, and what is out of scope:
  per-edge floors, a `prob_ceiling`, and a `core3d` `prob_floor`).

## Downstream
