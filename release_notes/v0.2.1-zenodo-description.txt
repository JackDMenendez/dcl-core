dcl_core is a Python library implementing the A=1 Discrete Causal
Lattice (DCL) framework, the substrate underlying the "Geometry First"
research series.  v0.2.1 is a single-feature patch release: the
prob_floor parameter on the continuous-amplitude engine.  The package
continues to ship two engines under a single import, side-by-side, so
downstream research can compare them directly.

The first engine, dcl_core.core, is a near-verbatim port of the
continuous-amplitude formulation from Paper I (Menendez, 2026,
Zenodo: 10.5281/zenodo.20078529).  Sessions carry complex Dirac
spinor amplitudes (psi_R on the RGB sublattice, psi_L on CMY),
evolve under a bipartite tick rule, and enforce A=1 through a post-hop
renormalisation.  The second engine, dcl_core.core3d, is the
integer-token reimplementation that enforces A=1 as an exact integer
identity (sum_x (N_RGB(x) + N_CMY(x)) == n_units, with no float
tolerance).

What is new in v0.2.1
---------------------

A single additive, opt-in feature: the prob_floor parameter on
dcl_core.core.CausalSession.  It is the continuous engine's
delta_p_min knob.

Per-site probability is p(x) = |psi_R(x)|^2 + |psi_L(x)|^2 (the Born
density of the bipartite spinor at lattice node x).  At end-of-tick,
any node with 0 < p(x) < prob_floor has both spinor components scaled
by the real factor sqrt(prob_floor / p(x)), so p(x) == prob_floor with
phase (and the psi_R/psi_L ratio) preserved; the joint state is then
renormalised so sum(p) == 1, so the A=1 axiom is never broken.  Nodes
with p(x) == 0 carry no phase and stay at zero -- the floor is the
minimum non-zero probability quantum, the continuous-engine analogue
of core3d's 1/N = delta_p_min granularity.  prob_floor=None (the
default) clamps nothing and is bit-for-bit identical to the v0.2.0
engine.

The feature exists to unblock the downstream dcl-delta-p-min
investigation: the continuous (core) column of its 4-cell grid
(experiment exp_09_dp_min_floor) maps delta_p_min onto prob_floor and
cannot run without it.

This is released as a PATCH, not a MINOR: prob_floor is purely opt-in
and backward-compatible (no re-exported symbol added or removed), and
the project is still pre-1.0.  It is NOT the v1.0.0 cut -- the formal
API freeze remains gated on dcl-delta-p-min finishing and co-releases
with Paper III.

Tests at release: 252 passed, 26 xfailed, 0 skipped (v0.2.0: 232
passed).  The +20 are the prob_floor contract suite
(tests/test_prob_floor_a1_consistency.py): validation, bit-for-bit
backward compatibility, exact per-site clamp, phase/ratio
preservation, empty-node-stays-empty, and A=1 after every tick across
a decade sweep of prob_floor.  A diagnostic,
experiments/exp_03_prob_floor_diag.py, sweeps the floor and confirms
max|A-1| ~ 1e-16 at every value.  The 26 xfails remain pre-existing
Paper I port drift inherited with the engine.

Install
-------
    pip install "git+https://github.com/JackDMenendez/dcl-core@v0.2.1"

For symbolic verification (Clifford algebra) or the GPU backend:
    pip install "dcl_core[sympy] @ git+https://github.com/JackDMenendez/dcl-core@v0.2.1"
    pip install "dcl_core[gpu]   @ git+https://github.com/JackDMenendez/dcl-core@v0.2.1"
